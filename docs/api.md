# API Reference 馃殌

The Web2APK backend exposes several REST endpoints for building, analyzing, and downloading APKs.

## 1. Build Endpoints

### `POST /build`
Starts a new build job.
- **Request Body (Multipart Form-Data)**:
    - `appName`: (String) The name of the resulting app.
    - `url`: (String) The URL to convert into an APK (if `mode=url`).
    - `packageName`: (String, Optional) The Android package name (e.g., `com.company.myapp`).
    - `mode`: (String) Either `url` or `html`.
    - `htmlContent`: (String, Optional) The raw HTML code to use as the app source (if `mode=html`).
    - `icon`: (File) Optional app icon (PNG/JPG).
- **Response**:
    - `jobId`: (UUID) Unique identifier for the job.

### `GET /build-events/:jobId`
Server-Sent Events (SSE) stream for real-time logs.
- **Events**:
    - `phase`: Current build phase (1-5).
    - `log`: Raw log message from Gradle or internal tools.
    - `progress`: Number (0-100) representing build progress.
    - `done`: Final success event with download link.
    - `error`: Error message if build fails.

## 2. Decompile Endpoints

### `POST /decompile`
Starts a new decompilation job.
- **Request Body (Multipart Form-Data)**:
    - `apk`: (File) The APK file to decompile.
- **Response**:
    - `jobId`: (UUID) Unique identifier for the job.

### `GET /download-zip/:jobId`
Downloads the source code bundle generated by a decompilation job.
- **Response**: A `.zip` file containing extracted Smali/Java and assets.

## 3. Download Endpoints

### `GET /download/:jobId`
Downloads the generated APK from a successful build job.
- **Response**: The `.apk` file.

### `GET /health`
Server health check.
- **Response**: `{ status: 'ok', version: '2.0.0' }`
